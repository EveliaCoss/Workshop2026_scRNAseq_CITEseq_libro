Qual devo escolher?
A Você não conhece bem R/RStudio, ou quer que tudo funcione sem configurar nada: use a máquina virtual. Vá direto para a Etapa 1A abaixo e depois para o checklist.
! Seu computador tem menos de 8 GB de RAM total: use a instalação local (Opção B) instead. A virtual machine needs RAM set aside for the guest Ubuntu system on top of whatever your own OS and other apps are using. On an 8 GB host, the VM alone will struggle, and the course's single-cell steps (PCA, clustering, plotting) will be slow or may run out of memory.
B Você já tem R/RStudio configurado ou tem experiência com instalação: use a instalação local (Opção B). It will generate the course's plots faster than the VM on the same hardware.

The máquina virtual is available in the pasta do Google Drive — procure a pasta VirtualBox — software subpasta. The file is a máquina virtual image (~8 GB). You will need virtualization software installed first — VirtualBox (free) works on Windows, Mac, and Linux.

O que já está feito para você The máquina virtual runs Ubuntu 24.04 with RStudio installed, every package from Block 0 já instalados (Seurat 5.x and the rest — see the package list in Step 4 below), and the course dataset already saved at data/gse149689_subset_3k.rds. 1. Instalar o VirtualBox
Baixe pelo pasta do Google Drive (instaladores para Apple Silicon, Apple Intel e Windows incluídos). Para Linux: baixe a versão para sua distribuição em virtualbox.org/wiki/Linux_Downloads.
2. Importar a imagem da VM
Abra o VirtualBox, escolha File > Import Appliance, selecione o arquivo baixado. Mantenha a alocação padrão de RAM/CPU, a menos que seu computador seja limitado — veja a nota abaixo.

Por que aparece "Ubuntu" se você não conhece Linux: o Ubuntu aqui é apenas o sistema operacional onde o RStudio roda. Você interagirá somente com o RStudio, da mesma forma que no Windows ou Mac. Você não precisa saber nenhum comando Linux para este curso.

Tudo já está instalado — então por que pode ser lento? Todos os pacotes e o dataset já estão lá, então você pula a etapa de instalação. Mas durante o curso, gerar gráficos (PCA, UMAP, clustering, os passos WNN do Block 4) ainda roda computação real dentro da VM. A velocidade depende apenas da RAM e dos núcleos de CPU que o seu computador pode ceder à VM — não de nada mais a instalar. Uma máquina com mais RAM e núcleos disponíveis renderizará os gráficos do curso visivelmente mais rápido.

Corrigir tela com resolução baixa ou cortada

Se a janela da VM abrir com resolução pequena ou estranhamente cortada, o botão de resolução da barra de ferramentas nem sempre funciona diretamente. Altere pelo menu de configurações:

Antes de iniciar a VM
No VirtualBox, selecione a VM, clique em Settings > Display, e aumente o Video Memory se a opção de alterar resolução estiver desabilitada.
Com a VM rodando
Dentro do Ubuntu, vá em Settings > Displays (ícone de engrenagem, não os controles do VirtualBox) e escolha uma resolução que corresponda à sua tela.

Alterar RAM e núcleos de CPU alocados para a VM

A VM roda tão rápido quanto os recursos que você dá a ela. Para alterar quanta RAM ou quantos núcleos de CPU ela pode usar:

Onde alterar Desligue a VM completamente primeiro (não pode ser redimensionada enquanto roda). No VirtualBox, selecione a VM na lista, clique em Settings, vá em System > Motherboard para alterar a RAM, e em System > Processor para alterar o número de núcleos de CPU. Como regra geral, deixe pelo menos 2-4 GB de RAM e um núcleo de CPU livres para o seu sistema operacional.
Se o seu computador tiver menos de 8 GB de RAM Do not increase the VM's RAM beyond what your host can spare, and consider using the instalação local (Opção B) instead — see the decision box above. Rodening the VM on a tight RAM budget will make every step that produces a plot noticeably slower, regardless of how the allocation is tuned.

Rodar o script de instalação
Instala do CRAN e Bioconductor com fallback automático
▼
Esta lista corresponde exatamente ao Block 0 do script do curso 14 pacotes, incluindo SingleCellExperiment, necessário para a etapa de detecção de doublets (scDblFinder) in Block 2. If you installed packages before from an older version of this guide, re-run the script below — it skips anything já instalados and only adds what is missing.
R — colar no console 
if (!requireNamespace("BiocManager", quietly = TRUE))
  install.packages("BiocManager")

all_pkgs <- c(
  "Seurat", "SeuratObject", "ggplot2", "patchwork",
  "dplyr", "Matrix", "harmony", "scales", "ggrepel",
  "SingleR", "celldex", "BiocParallel", "scDblFinder",
  "SingleCellExperiment"
)

for (pkg in all_pkgs) {
  if (!requireNamespace(pkg, quietly = TRUE)) {
    message("Installing: ", pkg)
    tryCatch(install.packages(pkg, quiet = TRUE),
             error = function(e) NULL, warning = function(w) NULL)
    if (!requireNamespace(pkg, quietly = TRUE)) {
      message("  Trying BiocManager...")
      tryCatch(
        BiocManager::install(pkg, ask = FALSE, update = FALSE),
        error   = function(e) message("  FAILED: ", e$message),
        warning = function(w) NULL
      )
    }
    if (requireNamespace(pkg, quietly = TRUE))
      message("  OK: ", pkg, " (", as.character(packageVersion(pkg)), ")")
    else
      message("  FAILED: ", pkg)
  } else {
    message("Already installed: ", pkg,
            " (", as.character(packageVersion(pkg)), ")")
  }
}

Visão geral do Block 1

O que o curso abrange

Este workshop foca em diagnosticar problemas reais em análises CITE-seq de célula única. Você rodará código que produz erros propositalmente e aprenderá a diagnosticá-los e corrigi-los.

scRNA-seq
Um perfil de RNA por célula. Cerca de 90% de zeros (dropout). Revela composição de tipos celulares, estados transcricionais e estrutura de população invisíveis ao sequenciamento bulk.
O CITE-seq adiciona
Medições de proteínas de superfície (ADT) sequenciadas junto com o RNA. Sinal bimodal, sem dropout. Equivalente a dados de citometria de fluxo para cada célula.
Por que o CITE-seq importa
O RNA de CD4 tem dropout intenso em verdadeiras células T CD4+. A proteína CD4 é limpa. Sem a camada de proteínas, a anotação de subtipos de células T depende de um sinal não confiável.
Contexto COVID-19
O dataset compara doadores saudáveis com pacientes COVID-19 em três níveis de gravidade. Expansão de monócitos e enriquecimento de plasmablastos são achados biológicos esperados.
O que esperar Alguns blocos de código produzem erros intencionalmente. Ao ver um erro, pare e leia o texto abaixo do bloco antes de continuar. Os erros são momentos pedagógicos, não enganos.
