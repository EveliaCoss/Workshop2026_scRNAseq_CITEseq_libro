Máquina virtual

Una máquina virtual Ubuntu 24.04 lista para usar con RStudio, el dataset y todos los paquetes del Block 0 ya instalados. Ideal si no usas R con frecuencia o no quieres resolver problemas de instalación en tu propio equipo.

Opción B

Instalación local

Instala R, RStudio y los paquetes directamente en tu computador. Más rápido durante el curso (sin sobrecarga de virtualización) y mejor si tu computador tiene poca RAM.

Pasos 2 a 5 abajo · 20-40 min en total
¿Cuál debo elegir?
A No conoces bien R/RStudio, o simplemente quieres que todo funcione sin configurar nada: usa la máquina virtual. Ve directamente al Paso 1A abajo y luego al checklist.
! Tu computador tiene menos de 8 GB de RAM total: usa la instalación local (Opción B) instead. A virtual machine needs RAM set aside for the guest Ubuntu system on top of whatever your own OS and other apps are using. On an 8 GB host, the VM alone will struggle, and the course's single-cell steps (PCA, clustering, plotting) will be slow or may run out of memory.
B Ya tienes R/RStudio configurado o tienes experiencia instalando paquetes: usa la instalación local (Opción B). It will generate the course's plots faster than the VM on the same hardware.
1A
Opción A — Download and run the máquina virtual
Ubuntu 24.04 · RStudio + paquetes + dataset preinstalados
▼

The máquina virtual is available in the carpeta de Google Drive — busca la carpeta VirtualBox — software subcarpeta. The file is a máquina virtual image (~8 GB). You will need virtualization software installed first — VirtualBox (free) works on Windows, Mac, and Linux.

VirtualMachine.amd64 — Intel & AMD VirtualMachine.arm64 — Apple Silicon & Snapdragon
Lo que ya está hecho para ti The máquina virtual runs Ubuntu 24.04 with RStudio installed, every package from Block 0 ya instalados (Seurat 5.x and the rest — see the package list in Step 4 below), and the course dataset already saved at data/gse149689_subset_3k.rds. Abres RStudio dentro de la VM y el script del curso está listo para ejecutar.
1. Instalar VirtualBox
Descarga desde el carpeta de Google Drive (instaladores para Apple Silicon, Apple Intel y Windows incluidos). Para Linux: descarga la versión para tu distribución desde virtualbox.org/wiki/Linux_Downloads.
2. Importar la imagen de la VM
Abre VirtualBox, elige File > Import Appliance, selecciona el archivo descargado. Mantén la asignación predeterminada de RAM/CPU salvo que tu equipo sea limitado — ver la nota abajo.

Por qué aparece "Ubuntu" si no conoces Linux: Ubuntu aquí es simplemente el sistema operativo donde corre RStudio. Solo interactuarás con RStudio, igual que en Windows o Mac. No necesitas saber ningún comando de Linux para este curso.

Todo está instalado — ¿por qué podría ser lento? Todos los paquetes y el dataset ya están ahí, por lo que te saltas la instalación. Pero durante el curso, generar gráficos (PCA, UMAP, agrupamiento, los pasos WNN del Block 4) aún ejecuta cálculo real dentro de la VM. La velocidad depende únicamente de la RAM y los núcleos de CPU que tu equipo puede ceder a la VM, no de nada más por instalar. Un equipo con más RAM y núcleos disponibles renderizará los gráficos notablemente más rápido.

Corregir pantalla con resolución baja o cortada

Si la ventana de la VM se abre con una resolución pequeña o extrañamente cortada, el botón de resolución de la barra de herramientas no siempre lo cambia directamente. Cámbialo desde el menú de configuración:

Antes de iniciar la VM
En VirtualBox, selecciona la VM, haz clic en Settings > Display, y aumenta la Video Memory si la opción de cambiar resolución aparece deshabilitada.
Con la VM en ejecución
Dentro de Ubuntu, ve a Settings > Displays (ícono de engranaje, no los controles de la ventana de VirtualBox) y elige una resolución que coincida con tu pantalla.

Cambiar la RAM y los núcleos de CPU asignados a la VM

La VM corre tan rápido como los recursos que le asignes. Para cambiar cuánta RAM o cuántos núcleos de CPU puede usar:

Dónde cambiarlo Apaga la VM completamente primero (no se puede redimensionar mientras corre). En VirtualBox, selecciona la VM en la lista, haz clic en Settings, ve a System > Motherboard para cambiar la RAM, y a System > Processor para cambiar el número de núcleos de CPU. Como regla general, deja al menos 2-4 GB de RAM y un núcleo de CPU libres para tu sistema operativo anfitrión.
Si tu computador tiene menos de 8 GB de RAM Do not increase the VM's RAM beyond what your host can spare, and consider using the instalación local (Opción B) instead — see the decision box above. Ejecutaning the VM on a tight RAM budget will make every step that produces a plot noticeably slower, regardless of how the allocation is tuned.

Ejecutar el script de instalación
Instala desde CRAN y Bioconductor con fallback automático
▼
Esta lista coincide exactamente con el Block 0 del script del curso 14 paquetes, incluido SingleCellExperiment, necesario para el paso de detección de dobletes (scDblFinder) in Block 2. If you installed packages before from an older version of this guide, re-run the script below — it skips anything ya instalados and only adds what is missing.
R — pegar en consola 
if (!requireNamespace("BiocManager", quietly = TRUE))
  install.packages("BiocManager")

all_pkgs <- c(
  "Seurat", "SeuratObject", "ggplot2", "patchwork",
  "dplyr", "Matrix", "harmony", "scales", "ggrepel",
  "SingleR", "celldex", "BiocParallel", "scDblFinder",
  "SingleCellExperiment"
)

for (pkg in all_pkgs) {
  if (!requireNamespace(pkg, quietly = TRUE)) {
    message("Installing: ", pkg)
    tryCatch(install.packages(pkg, quiet = TRUE),
             error = function(e) NULL, warning = function(w) NULL)
    if (!requireNamespace(pkg, quietly = TRUE)) {
      message("  Trying BiocManager...")
      tryCatch(
        BiocManager::install(pkg, ask = FALSE, update = FALSE),
        error   = function(e) message("  FAILED: ", e$message),
        warning = function(w) NULL
      )
    }
    if (requireNamespace(pkg, quietly = TRUE))
      message("  OK: ", pkg, " (", as.character(packageVersion(pkg)), ")")
    else
      message("  FAILED: ", pkg)
  } else {
    message("Already installed: ", pkg,
            " (", as.character(packageVersion(pkg)), ")")
  }
}

Vista previa del Block 1

Qué cubre el curso

Este taller se centra en diagnosticar problemas reales en análisis CITE-seq de célula única. Ejecutarás código que produce errores a propósito y aprenderás a diagnosticarlos y corregirlos.

scRNA-seq
Un perfil de RNA por célula. Alrededor del 90% de ceros (dropout). Revela composición de tipos celulares, estados transcripcionales y estructura poblacional invisible al secuenciamiento bulk.
El CITE-seq añade
Mediciones de proteínas de superficie (ADT) secuenciadas junto con el RNA. Señal bimodal, sin dropout. Equivalente a datos de citometría de flujo para cada célula.
Por qué importa el CITE-seq
El RNA de CD4 tiene un dropout intenso en verdaderas células T CD4+. La proteína CD4 es limpia. Sin la capa de proteínas, la anotación de subtipos de células T depende de una señal poco confiable.
Contexto COVID-19
El dataset compara donantes sanos con pacientes COVID-19 en tres niveles de gravedad. La expansión de monocitos y el enriquecimiento de plasmablastos son hallazgos biológicos esperados.
Qué esperar Algunos bloques de código producen errores intencionalmente. Cuando veas un error en pantalla, detente y lee el texto bajo el bloque antes de continuar. Los errores son momentos pedagógicos, no equivocaciones.
